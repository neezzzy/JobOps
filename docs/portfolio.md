## Case Study Copy

### Problem

Job seekers often manage applications across spreadsheets, browser tabs, notes, calendar reminders, and multiple resume files. That creates operational friction and makes it harder to track status, follow-ups, resume versions, and saved job posting context.

### Solution

JobOps organizes the job search as a local operations pipeline. Applications are grouped by status, follow-ups are tracked as workflow items, resume versions are saved with document metadata, and job posting details can be preserved for review.

### Technical Approach

The app uses Expo Router, React Native, TypeScript, and local SQLite persistence through `expo-sqlite`. The codebase separates screen UI, shared components, database access, migrations, services, types, and utilities. Services handle parsing helpers, recommendations, backup import/export, seed data, and document handling.

### Impact

JobOps demonstrates a practical local-first workflow tool that keeps personal job-search data on the device. It shows how operational dashboards and structured workflows can be applied to an everyday business process without relying on accounts, cloud sync, scraping, external AI APIs, paid APIs, or backend infrastructure.
